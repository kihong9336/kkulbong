## 2. 게임 개념 설명 (비개발자용)

### 쓰리매치란?
- 격자판에 여러 종류의 블록이 배치됨
- 플레이어가 인접한 두 블록을 **교환**
- 같은 종류의 블록이 **가로 또는 세로로 3개 이상** 나란히 오면 → **폭발(제거)**
- 제거된 자리는 위에서 블록이 내려오고, 새 블록이 위에서 채워짐
- 점수를 최대한 많이 쌓는 것이 목표

---

## 3. 핵심 기능 (MVP - 꼭 필요한 것만)

### 3.1 게임판
- 크기: **8행 x 8열** (64칸)
- 블록 종류: **6가지** (색상으로 구분: 빨강, 파랑, 초록, 노랑, 보라, 주황)
- 게임 시작 시 랜덤 배치 (단, 시작부터 3개 연속은 없도록)

### 3.2 블록 교환
- 블록을 클릭(또는 터치)해서 선택 → 인접 블록 클릭 시 교환
- 교환 후 3개 이상 매치가 없으면 → **교환 취소** (원위치)
- 교환 후 매치가 있으면 → 제거 + 점수 획득

### 3.3 블록 제거 & 채움
- 3개 일치: 제거
- 4개 일치: 제거 + 보너스 점수
- 5개 이상 일치: 제거 + 큰 보너스
- 제거 후 위에서 블록이 **떨어지는 애니메이션**
- 새 블록이 위에서 채워짐
- 연쇄 반응(콤보) 확인 후 추가 제거

### 3.4 점수 시스템
- 기본: 블록 1개 제거 = 10점
- 4개 이상: 추가 20점 보너스
- 콤보(연쇄): 콤보 수 x 10점 추가

### 3.5 타이머 & 게임 종료
- 제한 시간: **60초**
- 타이머가 0이 되면 게임 오버
- 최종 점수 표시 + 다시 시작 버튼

## 5. 개발 단계 (로드맵)

| 단계 | 내용 | 설명 |
|------|------|------|
| **1단계** | 기본 틀 | HTML 파일 만들기, 격자판 그리기 |
| **2단계** | 게임 로직 | 블록 교환, 매치 감지, 제거 구현 |
| **3단계** | 애니메이션 | 블록 떨어지기, 제거 효과 |
| **4단계** | 점수/타이머 | 점수 계산, 60초 타이머 |
| **5단계** | 마무리 | 게임 오버 화면, 디자인 다듬기 |
| **6단계** | 배포 | GitHub Pages에 올리기 |

## 7. 용어 정리 (비개발자용)

| 용어 | 의미 |
|------|------|
| MVP | Minimum Viable Product - 핵심 기능만 담은 최소 버전 |
| HTML | 웹페이지의 뼈대 (내용과 구조) |
| CSS | 웹페이지의 옷 (색상, 크기, 위치) |
| JavaScript | 웹페이지의 두뇌 (동작, 게임 로직) |
| 격자판(Grid) | 바둑판처럼 칸이 나뉜 게임판 |
| 매치(Match) | 같은 종류가 3개 이상 나란히 있는 상태 |
| 콤보 | 한 번의 교환으로 연쇄적으로 매치가 발생하는 것 |
| 배포 | 만든 게임을 인터넷에 올려 다른 사람이 접속할 수 있게 하는 것 |
